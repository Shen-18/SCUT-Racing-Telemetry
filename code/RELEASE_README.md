# SCUT Racing Telemetry — 使用手册

**SCUT Racing Telemetry** 是一款用于查看和对比 AiM 赛车遥测文件的 Windows 桌面工具。它支持读取 AiM 数据记录仪的原始文件（XRK/XRZ）以及 RaceStudio3 导出的 CSV 文件。

---

## 系统要求

- Windows 10 或 Windows 11
- 无需安装 Python 或其他运行时

## 快速开始

1. 将整个发布文件夹解压到任意位置
2. 双击 `SCUTRacingTelemetry.exe`
3. 在主页点击 **打开文件 A** 选择遥测文件开始分析

> **不要**将 `.exe` 单独移出文件夹，它依赖 `_internal/` 目录中的运行时组件和配置文件。

---

## 界面概览

### 主页（资料库）

用于导入和管理遥测文件。

- **导入文件**：点击顶部按钮导入单个文件、文件夹或 ZIP 压缩包
- **搜索过滤**：按文件名搜索已导入的记录
- **分组查看**：按日期、车手或车辆对记录进行分组
- **记录操作**：选中记录后可打开分析、导出 ZIP、编辑备注或删除
- **资料库**：所有导入的文件复制到 `library/` 目录，以 SHA-256 哈希去重

### 分析页

核心数据分析界面，分为三栏：

| 区域 | 功能 |
|------|------|
| 左侧通道栏 | 列出文件中所有数据通道，勾选即可在图表显示 |
| 中间图表区 | 渲染勾选通道的折线图（Time 为 X 轴） |
| 右侧统计栏 | 显示游标位置的各通道值、统计数据、对比指标 |

### 设置页

- 切换深色 / 浅色主题
- 调整显示预设（小 / 中 / 大字号）
- 打开配置文件 `setting.md` 进行高级设置

## 详细使用方法

### 打开单个文件

1. 点击左侧 **打开文件 A** 按钮
2. 选择 `.xrk`、`.xrz` 或 `.csv` 文件
3. 文件加载后，左侧会列出所有数据通道
4. **勾选通道**名称前的复选框以显示对应的折线图

支持的数据通道示例：
- `GPS Speed [km/h]` — GPS 速度
- `RPM [rpm]` — 发动机转速
- `TPS [%]` — 节气门开度
- `Water Temp [°C]` — 水温
- `Lambda [lambda]` — 空燃比

### 使用时间轴

**总览时间轴**位于图表区底部，用于控制主图表的时间范围：

- 拖拽总览轴上的**灰色选区**可平移可见时间窗口
- 拖拽选区**左右边缘**可缩小或扩大可见时间范围
- 主图表区跟随总览轴的变化自动更新
- 时间轴永远不能显示负值（Time ≥ 0）

### 读取特定时间点的值

- **单击**图表区域 → 放置一条红色竖线（游标）
- **按住鼠标左键拖动** → 游标跟随移动
- 游标所在位置的时间会显示在顶部
- 右侧面板实时显示游标处所有通道的值
- 红色竖线可以让你精确定位到某个时刻

### 双文件对比

比对两次跑动记录的数据，用于分析驾驶技术变化或调校效果。

1. 打开文件 A（点击 **打开文件 A**）
2. 点击 **打开文件 B** 选择第二个文件
3. 切换对比模式：
   - **叠图模式**：同一图表中绘制两条曲线，直接对比
   - **分图模式**：上下两个图表分别绘制，独立 Y 轴

#### 曲线对齐

由于两次跑动记录的开始时刻不同（例如一个从维修区出口开始，一个从发车线开始），需要将两条曲线在时间轴上对齐。

- **手动对齐**：拖动 **B 文件时间偏移**滑块，实时观察曲线对齐效果
- **自动对齐**：点击 **自动对齐** 按钮，软件基于互相关算法自动寻找最佳对齐偏移量

#### 对比指标

当选中通道且双文件模式激活时，右侧面板显示对比统计：
- **RMSE**：均方根误差（越小越相似）
- **MAE**：平均绝对误差
- **Corr**：皮尔逊相关系数（越接近 1 越相关）
- **Max Abs Error**：最大绝对误差

### 导出结果

| 操作 | 说明 |
|------|------|
| **导出 PNG** | 将当前图表区保存为 PNG 图片 |
| **导出当前视图 CSV** | 保存选定通道 + 当前时间窗口的数据 |
| **导出完整 CSV** | 导出完整数据为 RaceStudio3 兼容格式 |

## 配置说明

发布包附带 `setting.md` 文件，你可以用记事本打开它修改配置：

```ini
## 系统
default_theme = dark          # dark 或 light
display_preset = medium       # small / medium / large

## 文件
library_root = ./library      # 资料库存储路径
recursive_import = false      # 导入文件夹时是否递归
```

修改后**重启软件**生效。

## 技术说明

本软件使用以下技术构建：

### 核心栈
| 组件 | 用途 |
|------|------|
| PySide6 (Qt6) | 桌面用户界面框架 |
| pyqtgraph | 交互式时间序列图表绘制 |
| numpy + pandas | 数值计算与数据处理 |
| SQLite3 | 本地资料库存储 |
| PyInstaller | 打包为独立 Windows 可执行文件 |

### XRK 文件解析
对于 AiM 的 `.xrk` / `.xrz` 文件，本软件使用发布包中自带的**官方 AiM 解析器 DLL**（`MatLabXRK-2022-64-ReleaseU.dll`），通过 `ctypes` 技术桥接调用。它没有使用逆向工程的自定义解析器，因此兼容性完全对齐官方软件。

## 常见问题

**Q: 软件打不开 / 闪退？**
A: 请确保解压了完整文件夹，`_internal/` 目录和 `SCUTRacingTelemetry.exe` 必须在同级目录。

**Q: 支持中文文件名吗？**
A: 支持，建议使用 Windows 默认编码保存文件。

**Q: 如何更新软件版本？**
A: 备份你的 `library/` 文件夹，删除旧版本，解压新版本，然后将备份的 `library/` 放回。

**Q: 数据精度与 RaceStudio3 一致吗？**
A: XRK 文件通过官方 DLL 解析，数据完全一致。CSV 文件直接解析数值，精度受导出时保留小数位数影响。
